# PRレビュースキル

GitHubのプルリクエストをレビューし、LLMと議論するためのスキルです。
GitHubへの直接コメント投稿は行いません。

引数: `$ARGUMENTS`

---

## ステップ1: PR番号の特定

### 引数がある場合
`$ARGUMENTS` をPR番号として扱う。

### 引数がない場合

**必ず以下の手順を守ること。PR番号を直接尋ねることは禁止。**

#### ステップ1a: 一覧を表示する（スキップ禁止）

以下のコマンドを実行し、出力結果をそのままユーザーに見せる:

```bash
gh pr list --limit 30 --json number,title,author,headRefName \
  --template '{{range .}}#{{.number}}{{"\t"}}{{.title}}{{"\t"}}@{{.author.login}}{{"\n"}}{{end}}' \
  | awk -F'\t' '{printf "%-6s  %-55s  %s\n", $1, $2, $3}'
```

#### ステップ1b: 選択を待つ

一覧を表示した後、「レビューするPR番号を入力してください」と尋ねる。
ユーザーが番号を返したらステップ2へ進む。

---

## ステップ2: PR情報の収集

PR番号が確定したら、以下を**並列**で取得する:

```bash
# PR概要（本文・メタ情報）
gh pr view <PR番号> --json number,title,body,author,state,createdAt,updatedAt,labels,reviewDecision,additions,deletions,changedFiles,headRefName,baseRefName,commits

# PR上のコメント・レビューコメント一覧（作成日時付き）
gh pr view <PR番号> --json reviews,comments
```

---

## ステップ3: 過去のレビューノートの確認

リポジトリのスラッグを取得する:
```bash
gh repo view --json owner,name -q '"\(.owner.login)-\(.name)"'
```

ノートのパス: `~/.claude/pr-reviews/<repo-slug>/pr-<PR番号>.md`

ファイルが存在すれば読み込み、**最後にレビューした日時**（`最終レビュー:` の行）を抽出する。

---

## ステップ4: 表示モードの判断と表示

### 初回レビュー（過去ノートなし）

diff も取得する:
```bash
gh pr diff <PR番号>
```

以下の形式で全体を表示する:

```
## PR #<番号>: <タイトル>

**著者**: @<author> / **ブランチ**: <head> → <base>
**変更**: +<additions> / -<deletions>（<changedFiles>ファイル）

## 概要

<PR本文の要約 — 変更の目的・背景・アプローチを3〜5行でまとめる>

## 変更差分の要点

<diffを分析し、主な変更箇所を箇条書きで列挙する>

## コメント

<レビューコメント・一般コメントを時系列で列挙 — [@author] 要約>
```

### 再レビュー（過去ノートあり・最終レビュー日時が取得できる場合）

**前回レビュー日時以降**に追加されたコメント・レビューのみを対象にする。
新着がなければ「前回レビュー（<日時>）以降の更新はありません」と伝える。

diff は取得しない。ユーザーから「差分を見たい」「全体を見たい」と言われたときに初めて取得・表示する。

```
## PR #<番号>: <タイトル> — 前回レビュー以降の更新

**前回レビュー**: <日時>

## 新着コメント（<件数>件）

<前回レビュー日時より後のコメント・レビューを時系列で列挙 — [@author] 要約>
```

過去ノートは別セクションとして末尾に表示する（要約には含めない）:

```
---
### 📝 過去のレビューノート

<ノートの内容>
```

---

## ステップ5: 議論モード

表示後、ユーザーとの議論を開始する。

- 「差分を見たい」「全体概要を見たい」と言われたら、その時点で `gh pr diff` を取得して表示する
- ユーザーから「メモを残したい」「保存したい」という指示があった場合はステップ6へ

---

## ステップ6: レビューノートの保存（ユーザー指示時のみ）

`~/.claude/pr-reviews/<repo-slug>/pr-<PR番号>.md` に保存する。

ディレクトリが存在しない場合は `mkdir -p` で作成する。

既存ファイルがある場合は**追記**する（上書きしない）。

フォーマット（新規）:
```markdown
# PR #<番号> レビューノート

**PR**: <タイトル>
最終レビュー: <YYYY-MM-DD HH:MM>

## メモ

<ユーザーが残したい内容>

## 議論の要点

<議論の中で出た重要な指摘や決定事項>
```

フォーマット（追記）:
```markdown
---
最終レビュー: <YYYY-MM-DD HH:MM>

<新しい内容>
```

`最終レビュー:` の行はステップ3で日時を読み取るために使うため、必ず記載する。
